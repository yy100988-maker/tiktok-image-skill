# GPT-Image2 23 类模板详细说明

本文件是 `image` skill 对 GPT-Image2 风格库的中文操作指南。源库目前有 22 个具名模板；第 23 项是基于源库 `Other Use Cases` 分类整理的通用兜底模板，用于没有明确匹配项的特殊任务。

## 使用方法

先按“任务类别 -> 风格 -> 场景”选择模板，再写 提示词。每次 提示词 至少明确：主体与任务、画面结构、视觉风格、文字内容、比例和负面约束。商品参考图存在时，商品真实外形和可观察事实优先于风格。

## 1. UI 截图系统

**ID**：`ui-screenshot-system`

**适用**：App 截图、网站首页、后台仪表盘、社交动态、直播界面、产品 UI 原型。

**提示词 必填**：平台类型、设备尺寸、状态栏、导航结构、Tab、主要操作区、卡片层级、精确可见文字、交互状态和屏幕比例。

**视觉重点**：先锁定信息层级，再安排组件；主按钮、导航和内容区必须有明确优先级，文字必须短且可读。

**避免**：泛化的“现代 App 界面”、过多小字、平台特征混杂、无意义的按钮和无法读取的占位文字。

## 2. 信息图引擎

**ID**：`infographic-engine`

**适用**：知识卡片、产品卖点图、流程图、技术解释图、时间线、教育内容。

**提示词 必填**：主题、3-5 个信息模块、阅读顺序、箭头关系、图标、短标签、配色和画布比例。

**视觉重点**：模块数量少而清晰；用色块、编号、箭头和留白建立阅读路径。

**避免**：把长文章塞入图片、模块超过 6 个、所有信息同样醒目、标签过长。

## 3. 科学尺度示意图

**ID**：`scientific-scale-diagram`

**适用**：细胞、材料、器件、人体、宇宙等从微观到宏观的尺度对比。

**提示词 必填**：尺度起点和终点、单位、倍率、6-8 个尺度节点、局部放大窗口和短标签。

**视觉重点**：每个尺度必须有明显差异，倍率和单位要统一，放大窗口要指向具体结构。

**避免**：所有窗口长得一样、只画一个放大镜、缺少单位、尺度关系不连续。

## 4. 海报排版系统

**ID**：`poster-layout-system`

**适用**：活动海报、促销海报、展览海报、社交媒体封面、商业广告。

**提示词 必填**：主标题、次标题、日期地点、品牌标识、信息层级、版式网格、主视觉和文字占比。

**视觉重点**：先确定主标题的视觉权重，再安排副信息；标题、主体和行动指令要形成清晰动线。

**避免**：文字平均分布、标题不突出、信息堆叠、品牌和行动指令被背景吞没。

## 5. 运动商业海报

**ID**：`sports-campaign-poster`

**适用**：运动品牌、健身、户外、赛事、鞋服和能量型广告。

**提示词 必填**：运动项目、人物动作、速度方向、镜头焦段、汗水或尘土、品牌色、口号和比例。

**视觉重点**：动作必须有明确方向，使用低机位、动态透视、速度线或冻结瞬间表现力量。

**避免**：人体关节错误、动作不符合项目规律、运动员脸部或服装变化、过度使用速度特效。

## 6. 概念字体海报

**ID**：`conceptual-typography-poster`

**适用**：文字海报、概念展览、音乐活动、文化主题和社交传播视觉。

**提示词 必填**：精确文字、字体性格、字距、行距、文字变形方式、文字与主体的关系、留白区域。

**视觉重点**：让字体成为构图主体；明确哪些字必须原样出现，哪些内容仅作为装饰。

**避免**：文字乱码、过多字体、长句密集、把不可读的伪文字当作最终成品。

## 7. 水墨双重曝光海报

**ID**：`ink-double-exposure-poster`

**适用**：水墨人物、文化海报、诗意肖像、历史人物和传统意象融合。

**提示词 必填**：人物或主体、第二层意象、墨色层次、纸张纹理、构图方向、文字区域和文化语境。

**视觉重点**：两层图像要有明确关联，控制墨色浓淡和边缘扩散，保留纸面呼吸感。

**避免**：无关元素拼贴、传统符号乱用、人物五官被墨迹破坏、墨色过脏。

## 8. 自然科普海报

**ID**：`nature-science-poster`

**适用**：植物、动物、生态、自然保护、博物馆和科普传播。

**提示词 必填**：物种或自然对象、科学事实、观察视角、分类标签、环境层次和信息模块。

**视觉重点**：主体要可识别，标签与图像关系清楚，兼顾科学准确和视觉吸引力。

**避免**：混合错误物种特征、虚构科学数据、背景过度装饰、标签遮挡主体。

## 9. 商品商业视觉

**ID**：`product-commerce-visual`

**适用**：商品主图、详情页、广告素材、包装展示、卖点图和电商投放。

**提示词 必填**：产品真实外形、材质、颜色、视角、背景、光线、卖点、目标市场、文案区域和比例。

**视觉重点**：SKU 一致性高于风格；有参考图时锁定结构、配件、接口、比例和包装，只改变背景、光线或版式。

**避免**：换款、改结构、增加不存在的功能、虚构材质、把多张详情图合成一张拼图。

## 10. 个性化美妆报告

**ID**：`personalized-beauty-report`

**适用**：护肤分析、美妆推荐、肤色报告、产品组合和个性化美容内容。

**提示词 必填**：用户画像、观察到的肤色或妆容特征、产品类别、报告模块、推荐逻辑、免责声明和文案语言。

**视觉重点**：把报告做成可扫描的信息结构，人物、色板、产品和建议之间要有清晰对应关系。

**避免**：把猜测写成医学结论、虚构肤质问题、使用绝对化功效表述、过小的报告文字。

## 11. 品牌识别包

**ID**：`brand-identity-package`

**适用**：Logo 方向、品牌色、字体、名片、包装和基础视觉识别系统。

**提示词 必填**：品牌名称、行业、品牌性格、Logo 结构、色板、字体方向、应用物料和展示顺序。

**视觉重点**：先建立标志和色彩逻辑，再展示应用；同一品牌的所有触点必须保持一致。

**避免**：一次生成过多无关 Logo、品牌名拼写错误、颜色没有色值或层级、应用物料风格不统一。

## 12. 品牌触点视觉板

**ID**：`brand-touchpoint-board`

**适用**：品牌触点总览、包装、门店、社交媒体、广告和 campaign 灵感板。

**提示词 必填**：品牌规则、展示触点清单、统一色彩、字体、材质、镜头语言和画板布局。

**视觉重点**：这是一个统一视觉系统的展示板，不是随机物料拼贴；每个触点都要服务同一品牌印象。

**避免**：触点过多、画面像样片堆叠、品牌色漂移、各模块没有共同的视觉规则。

## 13. 建筑与空间

**ID**：`architecture-space`

**适用**：建筑外观、室内设计、门店、展厅、城市空间和概念场景。

**提示词 必填**：建筑类型、功能、材料、结构、光线时间、镜头位置、人物尺度和环境关系。

**视觉重点**：先确定空间动线和透视，再补材质与氛围；人物只用于提供尺度和使用状态。

**避免**：结构不可能、门窗比例错误、材质混乱、过度雾化导致空间不可检查。

## 14. 写实摄影

**ID**：`realistic-photography`

**适用**：商业摄影、人物肖像、手机照片、胶片质感、生活方式和真实场景。

**提示词 必填**：主体、摄影类型、相机或焦段、景别、光源、时间、构图、肤质或材质细节。

**视觉重点**：用可拍摄的摄影语言描述结果，例如自然窗光、35mm、浅景深、手持抓拍，而不是堆抽象形容词。

**避免**：塑料皮肤、过度锐化、虚假的镜头光斑、背景与主体光向不一致。

## 15. 街头纪实瞬间

**ID**：`street-accident-moment`

**适用**：街头纪实、突发事件、新闻感瞬间、城市观察和社会传播素材。

**提示词 必填**：事件性质、地点、人物动作、拍摄者位置、时间、环境反应、新闻构图和安全边界。

**视觉重点**：强调瞬间性和现场关系，保留不完美构图、自然反应和环境线索。

**避免**：血腥细节、煽动性误导、把虚构事件包装成真实新闻、人物身份和品牌信息的无依据断言。

## 16. 插画与艺术风格

**ID**：`illustration-art-style`

**适用**：插画、绘画、儿童内容、装饰图、材质实验和艺术风格探索。

**提示词 必填**：媒介、笔触、纸张或表面、调色板、构图、主体形状、细节密度和情绪。

**视觉重点**：指定可观察的材料和制作方式，例如木刻、彩铅、厚涂、版画或拼贴，而不是只写“高级艺术感”。

**避免**：风格词互相冲突、材质没有落到画面、细节过多破坏轮廓、模仿在世艺术家的个人签名风格。

## 17. 角色设定表

**ID**：`character-design-sheet`

**适用**：角色三视图、表情表、服装设定、姿态参考、游戏角色和影视前期设计。

**提示词 必填**：角色身份、年龄范围、体型、服装、配件、正侧背视图、表情数量、姿态和比例。

**视觉重点**：同一角色的脸型、发型、服装和配件要在所有视图中一致；图纸布局要有清晰标签。

**避免**：不同视图换脸、服装细节漂移、视图数量不明、把设定表做成单张海报。

## 18. 3D 收藏玩具

**ID**：`3d-collectible-toy`

**适用**：潮玩、盲盒、收藏公仔、手办、玩具包装和 3D 产品概念。

**提示词 必填**：角色或产品造型、材质、比例、关节、底座、包装、渲染引擎感、摄影棚和视角。

**视觉重点**：明确是实体玩具还是概念渲染，锁定材质、表面粗糙度、颜色和包装结构。

**避免**：多余肢体、不可生产结构、包装文字乱码、把玩具渲染成真人或软体生物。

## 19. 场景叙事

**ID**：`scene-storytelling`

**适用**：叙事场景、世界观、故事插图、分镜、直播画面和社交短内容。

**提示词 必填**：人物关系、事件前后、地点、时间、情绪、镜头位置、前中后景和叙事线索。

**视觉重点**：画面要能回答“谁在什么地方做什么”；用物件、姿态和光线暗示故事，而不是只靠大段文字。

**避免**：人物关系不清、没有事件动作、背景喧宾夺主、同一画面塞入多个不相关情节。

## 20. 历史与古风题材

**ID**：`history-classical-themes`

**适用**：历史人物、古风、传统文化、古典诗意、卷轴和博物馆视觉。

**提示词 必填**：朝代或时代、服饰制度、器物、建筑、社会身份、构图形式和文化情绪。

**视觉重点**：历史准确性优先；明确使用卷轴、册页、壁画或海报哪一种呈现方式。

**避免**：混用不同朝代、现代物品穿帮、虚构历史事实、传统符号只做表面装饰。

## 21. 文档与出版物

**ID**：`document-publishing`

**适用**：白皮书、手册、报告、百科图鉴、出版物页面和信息密集型版面。

**提示词 必填**：页面尺寸、栏数、页眉页脚、标题层级、表格、图表、图注、目录和阅读顺序。

**视觉重点**：建立稳定的出版物网格；文字、图表、标题和页码要对齐，保持可读性。

**避免**：小字过多、图表和图注错位、页边距不一致、把整本书压成一张无层级图片。

## 22. 概念产品拆解

**ID**：`concept-product-breakdown`

**适用**：产品研发板、爆炸图、结构拆解、组件关系、工业设计和技术展示。

**提示词 必填**：产品类型、组件清单、拆解方向、编号、材料逻辑、装配关系、剖面或爆炸距离和展示格式。

**视觉重点**：让组件关系和装配逻辑可检查；用编号、引线和分层结构表达技术信息。

**避免**：未定义的混合任务、组件数量过多、引线交叉、标签过长、虚构看不见的内部结构。

## 23. 通用特殊视觉任务

**ID**：`general-special-visual`

**性质**：这是本地整合文档提供的兜底模板，不是源库中的独立具名章节。

**适用**：实验性视觉、跨类别任务、特殊展示板、临时创意方向、无法归入前 22 类的需求。

**提示词 必填**：最终产物类型、主体、组件、标签、材料逻辑、构图规则、输出比例和验收标准。

**视觉重点**：先把任务边界写清，再选择最接近的 1-2 个源模板作为结构参考；不要把所有风格词堆在一起。

**避免**：类别不明、目标不明、既要海报又要 UI 又要产品拆解但没有优先级。

## 提示词通用模板

```text
模板 ID：<template-id>
目标：<这张图要解决什么问题>
主体：<产品、人物、空间或信息主题>
构图与版式：<景别、视角、网格、模块、阅读顺序>
视觉风格：<媒介、材质、色彩、光线、摄影或渲染语言>
文字：<必须出现的精确文字；没有文字则写无文字>
比例与输出：<1:1 / 4:5 / 9:16 / 16:9 等>
必须保留：<参考图中的结构、颜色、配件、比例或事实>
禁止出现：<乱码、虚构功能、错误结构、collage、grid、multi-panel 等>
```

## 与 VTeTech 生图的衔接

1. 先根据本文件和 `style-library.md` 选择模板并生成 提示词。
2. 有参考图时，先用 VTeTech 图像理解确认可观察事实和一致性约束。
3. 将最终 提示词 交给 `scripts/generate_image.py` 或批量脚本。
4. 每张图单独调用 VTeTech `gpt-image-2`，除非用户明确要求多格版式。
5. 生成后检查商品结构、文字可读性、模板版式和比例是否符合要求。
